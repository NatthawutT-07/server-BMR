# 🚀 คู่มือการอัปเดตโค้ดและฐานข้อมูลขึ้น DigitalOcean VPS

เมื่อคุณมีการแก้ไขโค้ด อัปเดต Library หรือเปลี่ยนโครงสร้างฐานข้อมูล (`schema.prisma`) บนเครื่องของคุณ (Windows) ต่อไปนี้คือขั้นตอน Step-by-Step เพื่อนำสิ่งใหม่ๆ เหล่านี้ขึ้นไปอัปเดตบนเซิร์ฟเวอร์ (VPS) ให้ระบบออนไลน์ของคุณใช้โค้ดตัวล่าสุดครับ

---

## 💻 1. นำโค้ดขึ้น Git (เครื่อง Local ของคุณ)

ก่อนอื่นต้องเอาการเปลี่ยนแปลงทั้งหมดเซฟลง Git และ Push ขึ้น Repository (เช่น GitHub / GitLab / Bitbucket)

1. เปิด Terminal ในโฟลเดอร์ `backend-BMR` บนเครื่องของคุณ
2. รันคำสั่งเพิ่มไฟล์ที่เปลี่ยนแปลงทั้งหมด:
   ```bash
   git add .
   ```
3. บันทึก (Commit) การเปลี่ยนแปลง พร้อมใส่ข้อความอธิบาย (เปลี่ยน `อัปเดตแก้ไข API` เป็นข้อความของคุณ):
   ```bash
   git commit -m "อัปเดตแก้ไข API และตารางข้อมูลล่าสุด"
   ```
4. ส่งโค้ดขึ้น Git:
   ```bash
   git push
   ```

*(หมายเหตุ: หากไม่มี Git ใช้การก๊อปปี้โฟลเดอร์ขึ้น VPS แทนก็ได้ผ่านโปรแกรมเช่น FileZilla หรือ WinSCP แนะนำให้ใช้ Git เพื่อความชัวร์และเร็ว)*

---

## 🌐 2. เข้าสู่ VPS ของคุณ

1. เปิด PowerShell หรือ Command Prompt บน Windows
2. ใช้คำสั่ง SSH เพื่อล็อกอินเข้าเซิร์ฟเวอร์ DigitalOcean (เปลี่ยน `bmr` เป็น Username และ `api.bmrpog.com` หรือ IP เป็นของคุณ):
   ```bash
   ssh bmr@api.bmrpog.com
   ```

---

## 📂 3. ดึงโค้ดล่าสุดลงมา (Pull Updates)

หลังจากล็อกอินเข้ามาบน VPS แล้ว ให้เข้าไปหาโฟลเดอร์โปรเจคของคุณ (โดยปกติอาจจะอยู่ที่ `~/apps/backend-BMR` หรือในโฟลเดอร์ที่มีไฟล์ `ecosystem.config.js`)

1. เข้าไปที่โฟลเดอร์โค้ดหลัก:
   ```bash
   cd ~/apps/backend-BMR
   ```
   *(แก้ไขพาธด้านบนให้ตรงกับที่คุณวางโปรเจคไว้บน VPS จริงๆ)*

2. ดึงโค้ดล่าสุดจาก Git ลงมาทับของเดิม:
   ```bash
   git pull origin main
   ```
   *(หาก Branch คุณชื่อ master หรือชื่ออะไรให้ปรับตรง main เป็นชื่อนั้น)*

---

## 📦 4. อัปเดต Library (หากเพิ่ม Library ใหม่)

หากในการแก้โค้ดรอบนี้ คุณมีการสั่ง `npm install แพ็กเกจใหม่` ต้องมารันลงใน VPS ด้วยเพื่อให้มันรู้จักรหัสผ่าน

```bash
npm install
```

---

## 🗄️ 5. อัปเดตโครงสร้างฐานข้อมูล (Database Migration)

**ข้อนี้สำคัญมาก**: ถ้าคุณมีการไปแก้ไฟล์ `prisma/schema.prisma` (เช่น เพิ่มตารางใหม่ เพิ่มคอลัมน์ใหม่) คุณต้องสั่งอัปเดตโครงสร้าง Database บนเซิร์ฟเวอร์จริง

รันคำสั่ง Migration 2 ตัวเลือก ดังนี้ (แนะนำแบบรัน Production):

**กรณีรันอัปเดตปกติ (Deploy Database):**
```bash
npx prisma generate
npx prisma migrate deploy
```
*(`migrate deploy` จะอัปเดตตารางที่มีการทำไฟล์ Migration ทิ้งไว้จากเครื่องคุณโดยไม่ลบข้อมูลเก่าทิ้ง)*

---

## 🔄 6. รีสตาร์ท Server ให้โหลดโค้ดตัวใหม่

เมื่อโค้ดใหม่มาหมด ฐานข้อมูลอัปเดตเสร็จแล้ว เราต้องสั่งให้ PM2 ปิดรันของเก่า และเปิดรันของใหม่

1. สั่งรีบู๊ตแอปด้วย PM2 (ถ้าคุณตั้งชื่อแอปใน PM2 ว่า `bmr-backend` ให้ใช้ตามนี้):
   ```bash
   pm2 restart bmr-backend
   ```
2. เช็คให้แน่ใจว่าสถานะขึ้นคำว่า **"online"** สีเขียว และไม่มีการสลับรีสตาร์ทรัวๆ:
   ```bash
   pm2 status
   ```

---

## 🔍 7. (ทางเลือก) ตรวจสอบความถูกต้อง

หากต้องการยืนยันว่าโค้ดใหม่ทำงานถูกต้อง หรืออยากรู้ว่ามี Error แบบเงียบๆ ที่รันติดตรงไหนไหม ให้เช็คล็อกที่กำลังรันจริงว่าไหลลื่นหรือไม่:

```bash
pm2 logs bmr-backend --lines 20
```

---

## ✨ สรุป Flow สั้นๆ ที่ต้องใช้ประจำ

ครั้งต่อไปเมื่อจะอัปเดต ทำแค่ 4 ลำดับง่ายๆ นี้:
1. `git push` จากเครื่องคุณ
2. SSH เข้า VPS
3. ไปที่โฟลเดอร์ `cd ~/apps/backend-BMR`
4. เรียง 3 คอมมานด์รวดเดียว: `git pull && npm install && npx prisma migrate deploy && pm2 restart bmr-backend`

## 🚨 การแก้ปัญหาเฉพาะหน้า (Troubleshooting)

### ปัญหา: \`error: Your local changes to the following files would be overwritten by merge:\`
ปัญหานี้เกิดจากใน VPS มีไฟล์บางไฟล์ที่ถูกแก้สดๆ แล้วเซฟ (เช่นแก้ใน Nano) ซึ่งขัดแย้งกับโค้ดตัวใหม่บน Git ถ้ามั่นใจว่าโค้ดบน Git ตัวใหม่**ถูกต้องที่สุดแล้ว** และเรายอมทิ้งสิ่งที่แก้สดบน VPS ไป ให้ทำตามลำดับนี้เพื่อดึงโค้ดใหม่ลงมาบังคับทับ:

1. สั่งซ่อนการแก้ไขสดๆ บน VPS ทิ้งไปก่อน (`git stash`)
   ```bash
   git stash
   ```
2. จากนั้นสั่ง Pull โค้ดใหม่ลงมาได้อย่างปกติ:
   ```bash
   git pull origin main
   ```
3. แล้วเริ่มขั้นตอน `npm install` และ `pm2 restart` ต่อไปได้เลยครับ
